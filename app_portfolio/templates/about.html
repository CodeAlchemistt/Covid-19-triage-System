CAPSTONE

THIS PROJECT

Ritaved was developed as my academy final project for a Machine Learning & Full-Stack Development course. The brief required a complete end-to-end system — from raw data to deployed web application.

I chose COVID-19 risk triage because it presented a genuine technical challenge: extreme class imbalance, real clinical stakes, and a dataset large enough to make model performance differences meaningful.

Building Ritaved taught me to navigate the gap between a well-performing Jupyter notebook and a production-quality web service — serialising models, designing REST APIs, handling async frontend state, and presenting data for non-technical users.

KEY LEARNINGS

  • ▸ Handling severely imbalanced datasets using SMOTE without leaking into the test split
  • ▸ Exposing a scikit-learn model via a Flask REST API with proper JSON serialisation
  • ▸ Building async UI updates with vanilla Fetch — no React required for the dashboard
  • ▸ Designing for clinical users: clear risk indicators, disclaimers, and accessible language
  • ▸ Choosing evaluation metrics (F1, AUC-ROC) that reflect real-world cost of false negatives
